# Ralph Army Handoff Prompt

Copy everything below the line into a new Claude Code session opened in the `rent-platform` directory.

You are the **orchestrator** for the rent-platform ralph army. Read `~/.claude/CLAUDE.md` for your full operating instructions, then read `roadmap-progress.json` for the task graph.

## Your mission

Execute the 15-task technical roadmap defined in `roadmap-progress.json`. The roadmap is organized into 5 waves with dependency chains. You dispatch worker subagents (ralphs) to write code and reviewer subagents to score it.

## Critical rules

### 1. Worktrees only — no exceptions

Every ralph runs in its own **isolated git worktree**. Never run a ralph on the main worktree. Never run two ralphs in the same worktree. This prevents file collisions between parallel agents.

```
git fetch origin main
git worktree add ../worktrees/tXX-task-name origin/main -b ralph/tXX-task-name
```

Use `isolation: "worktree"` on every Task tool call. Workers must never touch the main working tree.

### 2. Pull requests — not direct merges

Do NOT merge branches directly into main. Every completed task must be submitted as a **pull request** using `gh pr create`. The PR title should follow the format `feat(TXX): Task Name`. The PR body should summarize what the ralph built and include the review scores.

```bash
# After review passes, push and create PR — do NOT merge locally
cd ../worktrees/tXX-task-name
git push origin ralph/tXX-task-name
gh pr create --base main --head ralph/tXX-task-name \
  --title "feat(TXX): Task Name" \
  --body "$(cat <<'EOF'
## Summary
- [what the ralph built]

## Review Scores
- [paste |||REVIEW_SCORE||| block]

## Task Prompt
- See `prompts/tXX-task-name.md`

🤖 Generated by Ralph Army
EOF
)"
```

### 3. Iteration hard limit

No ralph loop may exceed **100 iterations**. If a ralph hasn't output `TASK_COMPLETE` by 100 iterations, kill it, log the state to `roadmap-progress.json`, and flag for human review.

### 4. Wave execution order

Respect the dependency graph in `roadmap-progress.json`. Tasks within the same wave can run in parallel (separate worktrees). Never start a task whose dependencies aren't complete.

- **Wave 1** (no deps, run in parallel): T01 CI/CD, T02 Test Infra, T03 Observability
- **Wave 2** (depends on T02): T04 Twilio Security, T05 Rate Limiting, T06 Arch Hardening
- **Wave 3** (depends on T02+T04): T07 Screening Compliance, T08 Listing Provider, T09 ACH Payments
- **Wave 4** (depends on T06): T10 Onboarding, T11 Tenant Portal, T12 Prompt Registry
- **Wave 5** (depends on T07+T08+T12): T13 Vendor Dispatch, T14 Lease Renewal, T15 Doc Intelligence

### 5. Review cycle

After a ralph outputs `TASK_COMPLETE`, dispatch a reviewer using `reviewers/review-template.md`. If all scores ≥ 9.5, push and create PR. If any score < 9.5, re-dispatch the ralph with feedback. Max 3 review cycles per task before flagging as blocked.
